Demo Q&A Cheat Sheet

Potential questions, quick answers, and code references.

Linked List Implementation

  • Q: What data structure do you use for the inventory?
    • A: Custom ordered singly linked list storing bookType*.
    • Code: orderedLinkedList.h.
  • Q: How do you keep the list ordered?
    • A: insert walks the list using compare (handles pointers by dereference) and places the new node in order.
    • Code: orderedLinkedList.h (insert, compare overload).
  • Q: How do you choose the sort field?
    • A: Static bookType::sortCode drives relational operators; changing it changes ordering logic.
    • Code: booktype.cpp (operators switch on sortCode).
  • Q: Can you return a hop count or node pointer from lookups?
    • A: Yes—lookUpBookHopCount returns hops from head; lookUpBookNodePtr returns the node pointer; UI still uses bookType*.
    • Code: invmenu.cpp (lookup helpers), orderedLinkedList.h (headPtr() accessor), invmenu.h.
  • Q: How is memory cleaned up?
    • A: Main iterates the list and deletes each bookType*; list destructor then frees nodes.
    • Code: mainmenu.cpp (cleanup loop), orderedLinkedList.h (destroyList).

Loading prog.out

  • Q: How do you parse prog.out?
    • A: loadData reads marker/value pairs (1–9) per record, trims, converts qty/wholesale/retail, constructs bookType, and inserts to keep order.
    • Code: mainmenu.cpp (loadData function).

Cashier Module

  • Q: How do you support multiple books on one receipt?
    • A: Parallel dynamic int arrays for indices and quantities; resize as needed; receipt totals over arrays.
    • Code: cashier.cpp (arrays bookIndices, purchaseQtys).
  • Q: How do you prevent overselling/negative stock with repeated adds?
    • A: Track quantity already in cart per title, cap input to remaining stock; block adding if cart already holds all available copies.
    • Code: cashier.cpp (qtyAlreadyInCart, remaining calculation, quantity prompt).
  • Q: How do users know they are in Cashier?
    • A: Bordered header: “Cashier Module • Active Checkout”, shows date and cart count.
    • Code: cashier.cpp (renderCashierScreen).
  • Q: Where does the date on the receipt come from?
    • A: Auto-generated current date (menu::currentDate()), no user prompt.
    • Code: menuutils.cpp/.h (currentDate), cashier.cpp uses it.
  • Q: What happens on confirmation?
    • A: Inventory quantities updated, thank-you message printed.
    • Code: cashier.cpp (confirm block).

Reports Module

  • Q: How is the Reports menu rendered?
    • A: Same bordered UI style with highlight; renderReportsMenuScreen.
    • Code: reports.cpp (menu renderer + loop).
  • Q: How do the reports paginate?
    • A: repListing 20 per page, repWholesale 10 per page, repRetail 15 per page; arrow/any key to advance with rubber-band wrap.
    • Code: reports.cpp (per-report loops and handlePaginationInput).
  • Q: How are headers/dates handled?
    • A: Headers include menu::currentDate() and page X of Y; columns sized for readability.
    • Code: reports.cpp (headers), menuutils.cpp (date helper).
  • Q: How are totals shown?
    • A: Wholesale and retail totals computed before pagination; printed on last page.
    • Code: reports.cpp (total computations in repWholesale/repRetail).

Searching & Editing Inventory

  • Q: How do searches work?
    • A: Case-insensitive substring match on title or ISBN; returns selectable matches, then bookInfo or selection pointer.
    • Code: invmenu.cpp (searchInventory, performSearch, lookUpBook).
  • Q: How do add/edit/delete affect ordering?
    • A: Adds use insert to maintain ordering; edits modify fields in place; deletes remove node and delete the book pointer.
    • Code: invmenu.cpp (addBook, editBook, deleteBook), orderedLinkedList.h (insert, remove).
